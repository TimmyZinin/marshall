2 Описание проблемы

Диспетчеры координируют рейсы через 4+ мессенджера одновременно (WhatsApp, Telegram, Viber, Max).

Хаос в каналах связи

Сообщения теряются между платформами, водитель не знает кто на смене

Нет контроля статусов

Прибытие, транзитное время, простой — всё на словах, нет записей

Документы теряются

Термограммы, фотоотчёты, ТСД забываются или задерживаются

Штрафы до 100%

Опоздание >4ч = 15%, >6ч = 30% (срыв), >24ч = 100%. Средний рейс 80 000 руб, штраф 30% = 24 000 руб

3 Решение: AI Listener (Sprint 0)

Пассивная AI-система, подключается к существующим чатам диспетчеров:

  • Слушает ВСЕ текстовые сообщения (нулевые изменения для людей)
  • Парсит структурированные данные через LLM (trip_id, маршрут, статус, заказчик, urgency)
  • Отображает алерты в дашборде
  • Аналитический дашборд с ключевыми метриками
Ключевой принцип: Ноль изменений для пользователей. Люди работают как раньше. AI просто наблюдает и структурирует.
НЕ входит в Sprint 0: Голос в текст (STT), интерактивный бот, чек-листы, кнопки статусов, контроль документов, мультиплатформа (Max, WhatsApp) — всё на платные спринты.

13 Ключевые решения
  1. Только пассивное прослушивание — никаких ответов бота
  2. PostgreSQL вместо SQLite — масштаб 500+ партнёров
  3. MiniMax M2.5 как основной LLM
  4. Нет голоса/STT — отложено
  5. Pluggable-адаптер для подключения (MTProto vs Bot API)
  6. FastAPI + Chart.js — лёгкий дашборд
  7. Docker Compose для деплоя
  8. Нет алерт-бота — алерты только в дашборде
  9. Деплой на RUVDS Тима — Marshall без доступа
  10. Нет интеграции АРМ/ТМС — Sprint 0 автономен
  11. DM включены — слушаем и директные сообщения
